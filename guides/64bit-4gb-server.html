CachyOS / Arch Linux

Deploy a high-throughput, containerized home server. By utilizing modern kernel features like io_uring and cgroups v2 alongside aggressive caching, this stack handles demanding microservices effortlessly.

Why This Stack

  • io_uring: A modern, highly efficient asynchronous I/O API in the Linux kernel. It provides a massive throughput boost for heavy I/O workloads like Docker databases and cache stores.
  • cgroups v2: The unified hierarchy for resource control in Linux. It's the modern standard for Docker, offering superior memory management and preventing runaway containers from crashing your server.
  • Traefik: A modern HTTP reverse proxy and load balancer that natively integrates with Docker. It automatically discovers containers and handles Let's Encrypt TLS routing on the fly.
  • Redis Cache: Deploying an in-memory data structure store drastically reduces the read load on PostgreSQL databases, ensuring web applications respond instantly.
  • BORE Scheduler: CachyOS's Burst-Oriented Response Enhancer (BORE) scheduler optimally distributes CPU time, preventing background server processes from starving each other during high-traffic spikes.

Prerequisites

  • A 64-bit (x86_64) PC with 4GB (or more) of RAM.
  • A blank USB Flash Drive (4GB or larger).
  • A secondary computer for flashing the ISO image.
  • A wired Ethernet connection to your local network.

Step 1: Download & Flash

Download the CachyOS Server/CLI ISO or a standard Arch Linux ISO. CachyOS is recommended as it pre-configures the BORE scheduler and performance repositories.

Download CachyOS ISO →

Flashing via Linux Terminal

bash
sudo dd if=cachyos-linux-x86_64.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot the server from the USB drive.
  2. If using CachyOS, launch the CLI installer. If using Arch, use the archinstall script.
  3. Profile Selection: Choose a minimal/server profile. Do not install a desktop environment.
  4. Partitioning: Select your main drive, wipe it, and choose XFS as your filesystem. XFS is highly optimized for Docker's overlay2 storage driver.
  5. Configure your network to use a static IP (or rely on a DHCP reservation from your router).
  6. Create your administrator user, set the root password, complete the installation, and reboot.

Step 3: Post-Install Optimization

Log in via SSH or physical terminal to verify kernel parameters.

1. Verify io_uring Status

Ensure the asynchronous I/O API is active to accelerate your database containers.

bash
cat /proc/sys/kernel/io_uring_disabled

This should output 0 or 2. If it outputs 1, io_uring is disabled globally.

2. Verify cgroups v2

Arch and CachyOS default to cgroups v2, but it is best to verify before configuring Docker.

bash
mount -l | grep cgroup2

If you see output indicating cgroup2 is mounted at /sys/fs/cgroup, your system is correctly configured.

Step 4: Install the App Stack

Install the core components: container runtime, reverse proxy, database, and cache layer.

bash
sudo pacman -Syu sudo pacman -S docker traefik postgresql redis

Enable Docker and Redis

bash
sudo systemctl enable --now docker sudo systemctl enable --now redis

Configure Traefik for Docker

Set up Traefik to automatically route traffic based on container labels.

bash
sudo nano /etc/traefik/traefik.yml

Add the following minimal configuration:

yaml
providers: docker: endpoint: "unix:///var/run/docker.sock" exposedByDefault: false entryPoints: web: address: ":80" websecure: address: ":443"
bash
sudo systemctl enable --now traefik

Configure Redis Memory Limit

To prevent Redis from starving PostgreSQL, cap its memory usage.

bash
sudo sed -i 's/^# maxmemory /maxmemory 512mb/' /etc/redis/redis.conf sudo sed -i 's/^# maxmemory-policy noeviction/maxmemory-policy allkeys-lru/' /etc/redis/redis.conf sudo systemctl restart redis

Step 5: Verification

Run these commands to ensure your high-performance services are active and communicating.

Verify Docker Resource Control

bash
sudo docker info | grep -i cgroup

Output should specify Cgroup Driver: systemd and Cgroup Version: 2.

Verify Redis Connectivity

bash
redis-cli ping

The server should respond with PONG.

Verify Traefik

bash
traefik version

Ensures the reverse proxy binary is installed and executable.

Troubleshooting

Permission Denied when running Docker commands

If you are not running as root, your user needs to be added to the Docker group. Run sudo usermod -aG docker $USER, then log out and completely log back in to apply the group permissions.

Traefik cannot see Docker containers

If Traefik isn't routing traffic to your containers, ensure the traefik user has permission to read the Docker socket. Additionally, because we set exposedByDefault: false in the YAML, you MUST add the label traefik.enable=true to your Docker compose files for Traefik to pick them up.

PostgreSQL / Redis crashes due to Out-of-Memory (OOM)

If 4GB isn't enough for your specific workloads, ensure you have configured a swap file or ZRAM. In Arch, you can quickly generate a swapfile: sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile.

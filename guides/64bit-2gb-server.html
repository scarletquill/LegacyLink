Arch Linux (Minimal)

Achieve maximum control and zero bloat. By utilizing a minimal Arch Linux installation, we dedicate every megabyte of your 2GB RAM strictly to hosting secure, high-performance database workloads.

Why This Stack

  • Arch Minimal Install: Arch gives you exactly what you install and nothing more. This zero-bloat foundation is ideal for servers where every megabyte counts.
  • XFS Filesystem: A robust, production-grade filesystem that PostgreSQL heavily favors for data integrity and parallel I/O handling on database workloads.
  • systemd Hardening: Utilizing systemd's built-in security directives (like PrivateTmp and NoNewPrivileges) automatically sandboxes exposed services.
  • certbot Automation: Seamlessly integrates with Nginx to issue, configure, and automatically renew free Let's Encrypt SSL/TLS certificates.
  • noop Scheduler: For modern, SSD-based legacy hardware, bypassing the kernel's complex I/O queue sorting minimizes CPU latency, speeding up direct disk writes.

Prerequisites

  • A 64-bit (x86_64) legacy PC with 2GB of RAM.
  • A blank USB Flash Drive (2GB or larger).
  • A secondary computer for flashing the Arch ISO image.
  • A wired Ethernet connection (strictly required for the net-installer).

Step 1: Download & Flash

Download the latest official Arch Linux ISO. The image acts as a live environment to construct your custom server from scratch.

Download Arch Linux ISO →

Flashing via Linux Terminal

bash
sudo dd if=archlinux-x86_64.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot the server from the USB drive. You will drop into a root command prompt.
  2. Ensure you have internet connectivity by pinging an external server (ping archlinux.org).
  3. Launch the guided installer by typing: archinstall.
  4. Set your keyboard layout, locale, and region.
  5. Drive & Partitioning: Select your target drive. Choose to format the drive, and explicitly select XFS as the filesystem.
  6. Profile Selection: Choose the minimal profile. Do not select a desktop environment.
  7. Configure network (NetworkManager is recommended), create a superuser account, and set the root password.
  8. Save the configuration and hit Install. Once finished, type reboot.

Step 3: Post-Install Optimization

Log into your newly installed minimal server to apply kernel and security optimizations.

1. Configure the noop Scheduler

If your server uses an SSD, tell the kernel to bypass legacy mechanical sorting.

bash
echo none | sudo tee /sys/block/sda/queue/scheduler

(Note: Modern kernels use "none" as the equivalent for the legacy "noop". Replace sda with your actual drive identifier).

2. systemd Service Hardening

Protect your server by sandboxing the Nginx web server using systemd override files.

bash
sudo systemctl edit nginx

Add the following directives between the commented boundaries:

systemd
[Service] PrivateTmp=true NoNewPrivileges=true ProtectSystem=strict ProtectHome=true

Step 4: Install the App Stack

Deploy the web server, database, and SSL automation tools.

bash
sudo pacman -Syu sudo pacman -S nginx postgresql certbot certbot-nginx

Initialize PostgreSQL

Before starting PostgreSQL, you must initialize its data directory as the postgres user.

bash
sudo -u postgres initdb -D /var/lib/postgres/data sudo systemctl enable --now postgresql

Nginx & Certbot Integration

Start the web server, then run certbot to automatically provision your SSL certificate.

bash
sudo systemctl enable --now nginx sudo certbot --nginx

(Follow the interactive prompts to secure your specified domain name. Certbot will automatically rewrite your Nginx config for HTTPS).

Enable the certbot timer to automate certificate renewals:

bash
sudo systemctl enable --now certbot.timer

Step 5: Verification

Confirm the core database and web services are securely running.

Verify PostgreSQL

bash
psql --version sudo systemctl status postgresql

Test Nginx Configuration

bash
sudo nginx -t

Ensure it reports syntax is OK and the test is successful, especially after certbot modifications.

Verify Certificates

bash
sudo certbot certificates

Lists all managed certificates and their expiration dates.

Troubleshooting

archinstall script fails or crashes

Ensure you are fully connected to the internet before launching the script. If downloading packages fails, exit the installer and run reflector --latest 5 --sort rate --save /etc/pacman.d/mirrorlist to update your mirror list to the fastest servers before trying again.

PostgreSQL fails to start (Job for postgresql.service failed)

This almost always means the database cluster was not initialized. Check /var/lib/postgres/data. If it's empty, run the initdb command exactly as shown in Step 4. Do not run initdb as root; it must be executed as the postgres user.

Certbot cannot complete the ACME challenge

Certbot requires ports 80 and 443 to be externally visible. Verify that your router is forwarding these ports to your Arch server's static IP address. Also, confirm your DNS A-record points to your router's public IP.

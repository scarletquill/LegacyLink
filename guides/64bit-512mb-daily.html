Alpine Linux + Sway

Achieve a hyper-efficient, modern daily driver on constrained hardware by combining Alpine's musl libc core with Sway, a next-generation Wayland compositor.

Why This Stack

  • Alpine Core & musl libc: Alpine's base system is aggressively stripped down. Using musl libc instead of glibc means the foundational OS consumes under 50MB of RAM upon boot.
  • Sway (Wayland Compositor): Wayland eliminates the overhead of the legacy X11 display server. Sway offers a tiling, keyboard-driven workflow that is incredibly light on memory and CPU.
  • F2FS (Flash-Friendly File System): If you are booting from an older USB drive or a cheap SSD, F2FS significantly reduces write amplification and improves performance over ext4.
  • Foot Terminal: A minimalist, fast, and lightweight Wayland-native terminal emulator perfect for restricted memory boundaries.
  • Mako: A lightweight Wayland notification daemon that operates without the massive dependencies of traditional desktop notification systems.

Prerequisites

  • A 64-bit (x86_64) legacy PC with 512MB of RAM.
  • A blank USB Flash Drive (1GB or larger).
  • A secondary computer for flashing the ISO image.
  • A wired Ethernet connection (highly recommended for the initial setup-alpine process).

Step 1: Download & Flash

Download the Alpine Linux Standard (x86_64) ISO from the official site. This provides a minimal bootable environment to start our custom build.

Download Alpine Standard (x86_64) ISO →

Flashing via Linux Terminal

bash
sudo dd if=alpine-standard-3.x.x-x86_64.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot your legacy hardware from the USB drive.
  2. At the `localhost login:` prompt, type root.
  3. Type setup-alpine to launch the interactive setup script.
  4. Configure your keyboard layout, hostname, and networking (DHCP is usually fine).
  5. When prompted for a mirror, select 1 for the fastest available.
  6. Choose your target disk (e.g., sda) and select sys for a permanent hard drive installation.
  7. Once the installation script finishes, type reboot and remove the USB drive.

Step 3: Post-Install Optimization

Log back in as root to apply filesystem and memory optimizations tailored for 512MB RAM.

1. Format Storage with F2FS (Optional)

If you have an additional Flash-based drive (like a secondary SSD or USB) for data storage, install F2FS tools and format it.

bash
apk add f2fs-tools mkfs.f2fs /dev/sdb1

2. Configure ZRAM with LZ4

On a 512MB system, ZRAM is essential. We will use the highly efficient LZ4 algorithm.

bash
modprobe zram echo lz4 > /sys/block/zram0/comp_algorithm echo 256M > /sys/block/zram0/disksize mkswap /dev/zram0 swapon /dev/zram0 -p 10

To persist ZRAM across reboots on Alpine, add it to your local startup script:

bash
echo "modprobe zram" >> /etc/local.d/zram.start echo "echo lz4 > /sys/block/zram0/comp_algorithm" >> /etc/local.d/zram.start echo "echo 256M > /sys/block/zram0/disksize" >> /etc/local.d/zram.start echo "mkswap /dev/zram0" >> /etc/local.d/zram.start echo "swapon /dev/zram0 -p 10" >> /etc/local.d/zram.start chmod +x /etc/local.d/zram.start rc-update add local default

Step 4: Install the App Stack

Next, we install our Wayland window manager and core applications.

1. Install Core Packages

bash
apk update apk add eudev setup-devd udev apk add sway foot links2 mako

2. Sway Config Basics

We must set up Sway for a non-root user. First, create a standard user (e.g., student) and add them to the video and input groups.

bash
adduser student adduser student video adduser student input

Log in as your new user, and copy the default Sway configuration to your home directory:

bash
mkdir -p ~/.config/sway cp /etc/sway/config ~/.config/sway/config nano ~/.config/sway/config

Inside the config file, change the default terminal from `alacritty` (if set) to `foot`.

Step 5: Verification

Check that your base system is ready and properly compressed.

Verify Sway Version

bash
sway --version

Ensure it outputs the installed Sway compositor version.

Verify Memory & ZRAM

bash
free -h

You should see your very low baseline RAM usage, and 256MB allocated to Swap (ZRAM).

Verify Filesystems

bash
df -T

If you mounted an F2FS partition, it will display f2fs under the Type column.

Troubleshooting

Sway fails to launch or errors out

Wayland requires proper user permissions. Ensure your user is added to the video, input, and seat groups. You may also need to install and start seatd (apk add seatd; rc-update add seatd; rc-service seatd start) depending on your Alpine version.

Cannot format to F2FS

If mkfs.f2fs returns a "command not found" error, ensure you have enabled the community repositories in /etc/apk/repositories, and run apk update && apk add f2fs-tools.

Links2 browser lacks graphical interface

By default, Links2 runs as a text browser in the terminal. To launch it in graphical mode within Sway, run it using the command links2 -g. Note that it will still look very rudimentary, but it keeps RAM usage incredibly low.

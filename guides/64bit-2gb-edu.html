Fedora (minimal) + GNOME-lite

Provide students with an enterprise-grade learning environment. By stripping Fedora down to its essentials and leveraging btrfs compression, we maximize performance on 2GB of RAM.

Why This Stack

  • Fedora Base: Fedora is widely used in academic and enterprise Linux environments, providing a highly stable, up-to-date, and secure foundation for student workstations.
  • GNOME-lite: A minimal installation of the GNOME environment avoids the massive bloat of the full suite, preserving the intuitive, modern interface without consuming the entirety of your 2GB memory budget.
  • btrfs + zstd: Fedora defaults to the btrfs filesystem. By enabling transparent `zstd` compression, we drastically reduce disk I/O bottlenecks and effectively expand usable storage on small legacy drives.
  • systemd-oomd: Fedora's integrated Out-Of-Memory daemon gracefully handles memory spikes. If a student opens too many browser tabs alongside LibreOffice, the system kills processes safely rather than hard-freezing.
  • Anki: The undisputed gold standard for spaced repetition study, essential for language, medical, and law students handling massive amounts of memorization.

Prerequisites

  • A 64-bit (x86_64) PC with 2GB of RAM.
  • A blank USB Flash Drive (4GB or larger).
  • A secondary computer for flashing the ISO image.
  • A reliable internet connection (required for the netinstall).

Step 1: Download & Flash

Download the Fedora Everything netinstall ISO. This specific image allows us to bypass the bloated default Workstation payload and install only what we need.

Download Fedora Everything ISO →

Flashing via Linux Terminal

bash
sudo dd if=Fedora-Everything-netinst-x86_64.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot from the USB drive and launch the Anaconda installer.
  2. Configure your Language, Keyboard, and Timezone.
  3. Software Selection: This is the most crucial step. Select Minimal Install or Custom Operating System. Do NOT select the standard Fedora Workstation group.
  4. Installation Destination: Choose your hard drive. In the partitioning tool, select Automatic but ensure the filesystem type is set to Btrfs (Fedora's default).
  5. Create your student/admin user account and set a strong root password.
  6. Begin the installation. Once complete, reboot into the terminal.

Step 3: Post-Install Optimization

Log in to the command line as root (or use sudo) to set up memory and storage compression.

1. Enable btrfs zstd Compression

Modify the filesystem mount options to enable transparent compression, speeding up read/write times on older hard drives.

bash
sudo mount -o remount,compress=zstd / sudo nano /etc/fstab

In your /etc/fstab file, look for the lines mounting your / and /home btrfs subvolumes. Add compress=zstd to the options list (e.g., defaults,compress=zstd,subvol=root).

2. Ensure ZRAM is Active

Fedora enables ZRAM by default, but we should verify the generator is actively running to protect our 2GB boundary.

bash
sudo systemctl enable --now zram-generator

Step 4: Install the App Stack

Now we install our minimal GUI, office suite, and study tools using Fedora's dnf package manager.

bash
sudo dnf upgrade --refresh sudo dnf install @gnome libreoffice anki

Note: Using the @gnome group installs the base GNOME desktop without the massive array of extra Fedora Workstation default apps (like GNOME Maps, Weather, Boxes, etc.), keeping the footprint lean.

Set Graphical Target

Because we started from a minimal install, we need to tell systemd to boot into the graphical interface by default.

bash
sudo systemctl set-default graphical.target sudo reboot

Step 5: Verification

Confirm the system optimizations and educational tools are properly deployed.

Verify btrfs Compression

bash
sudo btrfs filesystem show mount | grep btrfs

The mount command output should display compress=zstd in the options list for your root volume.

Verify Memory Constraints

bash
free -h

Check the output. You should see your ZRAM active as the primary Swap space, buffering the 2GB physical limit.

Check Anki Installation

bash
anki --version

Ensures the spaced-repetition software is installed and ready for flashcard imports.

Troubleshooting

System boots to a blank screen or terminal

If you completed the installation but do not see the GNOME login screen, verify that the display manager (GDM) is enabled. Log into the terminal and run sudo systemctl enable --now gdm.

GNOME Software center is missing

Because we used the minimal `@gnome` group instead of the full workstation payload, the graphical software store is omitted to save RAM. If students desperately need a graphical way to install apps, you can add it via sudo dnf install gnome-software, though terminal usage is recommended for 2GB systems.

Anki sync is failing

Ensure the system time is perfectly synced. AnkiWeb relies heavily on accurate timestamps to merge study decks. Go to GNOME Settings → Date & Time, and ensure "Automatic Date & Time" is toggled on.

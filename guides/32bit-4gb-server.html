32-bit CPU | 4GB+ RAM | Home Server

Devuan + Docker-ce

Capitalize on the 4GB RAM sweet spot by turning legacy 32-bit hardware into a modular, containerized application host protected by Nginx and fail2ban.

Why This Stack

  • 4GB Docker Sweet Spot: 4GB provides sufficient overhead to run multiple microservices via Docker concurrently on 32-bit hardware without risking Out-Of-Memory (OOM) kernel panics.
  • cgroups v1 Requirement: Older x86 kernels and 32-bit Docker versions require cgroups v1 for resource management. Devuan makes it easy to specify this at the bootloader level.
  • XFS Filesystem: Crucial for Docker's overlay2 storage driver. XFS handles layered file systems significantly better than ext4, especially when spinning up and tearing down containers.
  • Nginx Reverse Proxy: A lightweight gateway that safely exposes multiple internal Docker containers to the outside world over single standard ports (80/443).
  • fail2ban: An automated intrusion prevention framework that bans malicious IPs, an essential security layer for any exposed server infrastructure.

Prerequisites

  • A 32-bit (x86) legacy PC with 4GB (or more) of RAM.
  • A blank USB Flash Drive (4GB or larger).
  • A secondary computer for flashing the ISO image.
  • A hardwired Ethernet connection (Wi-Fi is discouraged for container hosts).

Step 1: Download & Flash

Download the Devuan Chimaera 32-bit (i386) Netinstall ISO. We use the netinstall to maintain a headless, server-optimized base system devoid of GUI bloat.

Download Devuan i386 Netinstall ISO →

Flashing via Linux Terminal

bash
sudo dd if=devuan_chimaera_4.0.0_i386_netinstall.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot the target machine from the USB drive and choose Install.
  2. Proceed through the initial prompts to set up your network, hostname, and root password.
  3. Partitioning: Choose "Manual". Format the primary partition using the XFS filesystem, which is highly recommended for Docker.
  4. Init System: Explicitly select runit when prompted.
  5. Software Selection: Uncheck the desktop environment. Keep "SSH server" and "standard system utilities" selected.
  6. Finish the installation, install the GRUB bootloader, and reboot.

Step 3: Post-Install Optimization

Log in to your server. We must configure the kernel to allow Docker to properly isolate memory and swap accounts.

1. Configure cgroups v1

Docker requires specific kernel parameters on 32-bit architectures. Edit the GRUB configuration file:

bash
sudo nano /etc/default/grub

Find the line starting with GRUB_CMDLINE_LINUX and modify it to look exactly like this:

text
GRUB_CMDLINE_LINUX="cgroup_enable=memory swapaccount=1"

Save the file and update the bootloader:

bash
sudo update-grub sudo reboot

(The server must be rebooted for these kernel parameters to take effect.)

2. XFS Formatting (Secondary Drive)

If you have a dedicated drive for Docker containers (e.g., `/dev/sdb`), format it to XFS now and mount it to `/var/lib/docker`.

bash
sudo mkfs.xfs -f /dev/sdb1

Step 4: Install the App Stack

Fetch the Docker engine, the Nginx proxy, and the fail2ban security utility.

bash
sudo apt update sudo apt install docker-ce nginx fail2ban

Configure Nginx Reverse Proxy

Create a proxy configuration to route incoming traffic to a container (assuming a container is running on port 8080).

bash
sudo nano /etc/nginx/sites-available/default

Add the proxy block inside the server configuration:

nginx
location /app { proxy_pass [http://127.0.0.1:8080](http://127.0.0.1:8080); proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; }

Configure fail2ban

Create a local jail configuration to protect SSH and Nginx.

bash
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local sudo service fail2ban restart

Step 5: Verification

Run these commands to ensure the entire stack is operational.

Test Docker Engine

bash
sudo docker run hello-world sudo docker ps -a

The first command downloads and runs a test container. The second verifies the daemon is tracking it.

Verify Nginx Configuration

bash
sudo nginx -t

Output should say "syntax is ok" and "test is successful".

Check fail2ban Status

bash
sudo fail2ban-client status

Output will display the active jails (e.g., sshd) currently protecting the system.

Troubleshooting

Docker daemon fails to start

If Docker refuses to start, it is highly likely the cgroups memory configuration was not applied properly. Double-check your /etc/default/grub file for typos, ensure you ran update-grub, and confirm the machine was completely rebooted.

"exec format error" when running containers

You are using a 32-bit (i386) host. You cannot run modern 64-bit (amd64) Docker containers. Always check Docker Hub for the linux/386 architecture tag before attempting to pull and run an image.

Nginx returns 502 Bad Gateway

A 502 error means Nginx is working, but the Docker container it's trying to talk to is either down or listening on a different port. Verify your container is actively running using docker ps and check that the mapped ports align with your proxy_pass IP/port.

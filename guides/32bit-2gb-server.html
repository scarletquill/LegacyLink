32-bit CPU | 2GB RAM | Home Server

Devuan Chimaera

Deploy a stable, systemd-free server environment using the lightweight runit init system and a classic LEMP stack, optimized for low overhead.

Why This Stack

  • Systemd-Free (runit): Devuan replaces the monolithic systemd with runit, which is drastically lighter, more predictable, and consumes far fewer background resources on a 2GB machine.
  • Hardened Kernel: Implementing a hardened kernel profile reduces the attack surface on an always-on home server exposed to your local network.
  • No Swap Policy: Older HDDs suffer from mechanical wear during swap thrashing. With 2GB of RAM, we disable swap entirely, forcing the kernel to manage active memory aggressively without disk latency.
  • LEMP Stack (Nginx + MariaDB + PHP-FPM): The absolute standard for high-performance, low-memory web hosting. Nginx handles static requests asynchronously, while PHP-FPM scales worker processes dynamically.

Prerequisites

  • A 32-bit (x86) legacy PC with 2GB of RAM.
  • A blank USB Flash Drive (4GB or larger).
  • A secondary computer for flashing the ISO image.
  • A wired Ethernet connection (essential for server stability).

Step 1: Download & Flash

Download the Devuan Chimaera (i386 Netinstall) ISO. The netinstall ensures you only download the core packages required, avoiding desktop bloat.

Download Devuan i386 Netinstall ISO →

Flashing via Linux Terminal

bash
sudo dd if=devuan_chimaera_4.0.0_i386_netinstall.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot the server machine from the USB drive and select Install.
  2. Follow the Debian-style prompts to configure your language, timezone, and network.
  3. Set a strong root password and create an administrative user account.
  4. Partitioning: Choose "Manual" or "Guided - use entire disk". CRITICAL: Do not create a swap partition. Format the primary partition as ext4.
  5. Init System: When prompted to select your init system, explicitly choose runit.
  6. Software Selection: Uncheck "Debian desktop environment" and any display managers. Only leave "SSH server" and "standard system utilities" checked.
  7. Install the GRUB bootloader, reboot the system, and remove the USB drive.

Step 3: Post-Install Optimization

Log in via SSH or directly on the machine to apply the server optimizations.

1. Enforce No Swap Policy

If a swap file/partition was accidentally created during install, disable it permanently.

bash
sudo swapoff -a sudo sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab

2. Hardening: Disable Unused Services

Use Devuan's update-rc.d to remove unnecessary startup services, freeing up memory.

bash
sudo update-rc.d -f bluetooth remove sudo update-rc.d -f cups remove sudo update-rc.d -f avahi-daemon remove

Step 4: Install the App Stack

Update your package lists and install the LEMP stack.

bash
sudo apt update sudo apt install nginx mariadb-server php-fpm

Secure MariaDB

Run the built-in security script to remove test databases and lock down root access.

bash
sudo mysql_secure_installation

(Press 'Y' for all prompts to apply secure defaults).

Configure Nginx Virtual Host

Link Nginx to PHP-FPM by editing the default site block.

bash
sudo nano /etc/nginx/sites-available/default

Uncomment the PHP location block so it looks similar to this:

nginx
location ~ \.php$ { include snippets/fastcgi-php.conf; fastcgi_pass unix:/var/run/php/php7.4-fpm.sock; }

(Note: Your PHP version number might differ. Adjust php7.4-fpm.sock to match your installed version). Restart Nginx with sudo service nginx restart.

Step 5: Verification

Ensure the core web services are actively running and responding.

Check Nginx Service Status

bash
sudo service nginx status

(Because Devuan uses runit/sysvinit instead of systemd, we use service rather than systemctl. Ensure it reports as 'active' or 'running').

Test the Web Server Locally

bash
curl localhost

You should see the HTML output of the default "Welcome to nginx!" page.

Troubleshooting

systemctl command not found

Remember, Devuan explicitly removes systemd. If you are following standard Debian tutorials, replace systemctl restart [service] with sudo service [service] restart or use runit's native sv command.

PHP files are downloading instead of executing

This means Nginx is not passing PHP files to the PHP-FPM processor. Double-check your /etc/nginx/sites-available/default configuration. Ensure the fastcgi_pass socket perfectly matches the version in /var/run/php/.

MariaDB connection refused for non-root users

By default, MariaDB might use unix_socket authentication for root. If you need a web app to connect to the database, log in as root (sudo mysql) and create a dedicated database user with a standard password using GRANT ALL ON dbname.* TO 'user'@'localhost' IDENTIFIED BY 'password';

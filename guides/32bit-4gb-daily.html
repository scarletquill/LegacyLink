AntiX + Openbox

Push 32-bit architecture to its absolute limit. With 4GB of RAM, we drop the desktop environment entirely for the standalone Openbox window manager, giving maximum resources to modern applications.

Why This Stack

  • Openbox Comfort: With 4GB of RAM available, a pure Openbox window manager session allows you to comfortably run multiple applications and browser tabs simultaneously without UI lag.
  • ZRAM at 30%: At this tier, memory compression acts as a safety net to prevent hard drive swap during extreme multi-tasking, rather than a strict necessity for basic functioning.
  • XFS Filesystem: Outperforms ext2/ext4 on mechanical drives when handling larger files and parallel data streams, making the most of your 4GB breathing room.
  • PCManFM: A blazing-fast GTK-based file manager that perfectly complements Openbox, handling desktop icons and volume management seamlessly.
  • Pale Moon: Continues to be the optimal choice for legacy systems, providing the best actively developed 32-bit browser experience.

Prerequisites

  • A 32-bit (x86) legacy PC with 4GB (or more) of RAM.
  • A blank USB Flash Drive (4GB or larger).
  • A secondary computer for flashing the ISO image.
  • A wired Ethernet connection (recommended for post-install setup).

Step 1: Download & Flash

Download the antiX-full (32-bit) ISO. We will start with the base AntiX installation and strip it down to a pure Openbox environment.

Download AntiX 32-bit ISO →

Flashing via Linux Terminal

bash
sudo dd if=antiX-23_386-full.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier, e.g., /dev/sdb. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot the target machine from the USB drive.
  2. Click the Install icon on the live desktop.
  3. When you reach the partitioning section, select Custom Install to manually configure your filesystems.
  4. Format your root (/) and home (/home) partitions using the XFS file system instead of the default ext4.
  5. Complete the user setup and wait for the installation to finish.
  6. Reboot the system and log in.

Step 3: Post-Install Optimization

Open a terminal to apply the filesystem and memory optimizations.

1. Format Additional Drives to XFS (Optional)

If you have a secondary data drive, format it to XFS to leverage faster large-file handling.

bash
sudo mkfs.xfs -f /dev/sdb1

2. Configure 30% ZRAM

Set up ZRAM to compress roughly 1.2GB of your 4GB RAM footprint.

bash
sudo modprobe zram sudo zramctl /dev/zram0 --size 1.2G sudo mkswap /dev/zram0 sudo swapon /dev/zram0 -p 10

Make this persistent by adding the commands (without sudo) to /etc/rc.local.

3. Openbox Autostart Configuration

Prepare the local Openbox configuration folder for your user account.

bash
mkdir -p ~/.config/openbox touch ~/.config/openbox/autostart chmod +x ~/.config/openbox/autostart

Step 4: Install the App Stack

Fetch the dedicated application stack via APT.

bash
sudo apt update sudo apt install openbox pcmanfm palemoon-bin

Configure Desktop Management

Edit your autostart file to launch PCManFM in daemon mode so it manages your desktop wallpaper and icons.

bash
echo "pcmanfm --desktop &" >> ~/.config/openbox/autostart

(Logout of the current AntiX desktop and select "Openbox" from the session manager at the login screen).

Step 5: Verification

Ensure your storage and memory layers are functioning correctly.

Verify XFS Filesystem

bash
df -T

Check the "Type" column for your main partitions; they should display xfs.

Verify ZRAM and Memory

bash
free -h

You should see your total ~4GB of RAM, plus the ~1.2GB of Swap (which is your compressed ZRAM).

Troubleshooting

Blank Screen / No Menu in Openbox

Openbox is a blank slate window manager. If you haven't set a wallpaper with PCManFM or Nitrogen, it will just be a black or gray screen. Right-click anywhere on the background to bring up the root application menu.

PCManFM isn't managing the desktop

If your desktop is unresponsive to right-clicks (meaning PCManFM isn't controlling it), ensure you actually logged into the "Openbox" session and not "Openbox-Session" or another AntiX default. Verify your ~/.config/openbox/autostart file is executable.

Missing Volume / Network Applets

A pure Openbox setup requires you to launch your own system tray applets. You can install volumeicon-alsa and cmst (for Connman networking), then add them to your autostart file: volumeicon & and cmst -m &.

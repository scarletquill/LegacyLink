MX Linux + Xfce

Empower students with a fully-featured desktop environment. Leveraging 4GB of RAM, we utilize the Xfce desktop and the preload daemon to deliver a highly responsive educational workstation capable of handling design tools and virtual labs.

Why This Stack

  • Xfce Desktop Environment: Provides a traditional, full-featured desktop experience that is vastly lighter than GNOME or KDE, leaving the bulk of your 4GB RAM free for intensive applications.
  • Preload Daemon: Since we have 4GB of RAM to spare, we install preload, an adaptive background service that learns usage patterns and caches frequently used applications into memory for near-instant startup times.
  • GIMP: A professional-grade image manipulation program essential for art, design, and media students.
  • VirtualBox: Allows computer science students to run isolated lab environments or secondary operating systems (like a minimal Linux server) directly on their workstation.
  • XFS Filesystem: When dealing with large, contiguous files like VirtualBox disk images (.vdi), XFS dramatically outperforms ext4 in read/write operations and file allocation.

Prerequisites

  • A 32-bit (x86) legacy PC with 4GB (or more) of RAM.
  • A blank USB Flash Drive (4GB or larger).
  • A secondary computer for flashing the ISO image.
  • A stable internet connection.

Step 1: Download & Flash

Download the standard MX Linux 32-bit (Xfce) ISO from their official repository. This is the flagship edition and provides the best out-of-the-box experience.

Download MX Linux 32-bit ISO →

Flashing via Linux Terminal

bash
sudo dd if=MX-23_386.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot the legacy machine from the prepared USB drive.
  2. Launch the graphical installer from the live desktop.
  3. Critical Partitioning Step: When prompted to configure your partitions, choose Custom Install. Assign your root (/) and home (/home) partitions to use the XFS filesystem instead of the default ext4. This is vital for VirtualBox performance.
  4. Proceed through the remaining setup screens to create your student/admin user account.
  5. Reboot the system and remove the USB drive.

Step 3: Post-Install Optimization

We will apply XFS for external storage, initialize ZRAM, and set up the preload daemon.

1. Format VirtualBox Drive to XFS (If applicable)

If you have a secondary hard drive strictly for storing VM images, format it to XFS.

bash
sudo mkfs.xfs -f /dev/sdb1

2. Enable Preload Daemon

Install the preload utility to intelligently utilize your 4GB of RAM for application caching.

bash
sudo apt update sudo apt install preload sudo systemctl enable preload sudo systemctl start preload

3. Setup 30% ZRAM

Use the built-in MX Tools to enable memory compression as a safety net.

  1. Open MX Tools from the application menu.
  2. Click MX Tweak and navigate to the Performance tab.
  3. Enable ZRAM and set it to roughly 30% of your total RAM (~1.2GB).

Step 4: Install the App Stack

Use the terminal to quickly fetch the advanced educational suite.

bash
sudo apt install libreoffice gimp virtualbox

Setting up VirtualBox for Labs

To allow your standard user to utilize USB devices and network bridging inside VirtualBox, you must add them to the vboxusers group.

bash
sudo usermod -aG vboxusers $USER

(You must log out and log back in for this group change to take effect. When creating a new VM in VirtualBox, remember to allocate no more than 1GB of RAM to the guest OS to prevent starving the host).

Step 5: Verification

Verify that your background optimizations and heavy applications are correctly deployed.

Check Preload Status

bash
sudo systemctl status preload

Output should indicate the service is active and running.

Verify VirtualBox

bash
vboxmanage --version

Output will display the installed version of VirtualBox.

Troubleshooting

VirtualBox cannot run 64-bit VMs

Because you are installing this on a 32-bit host operating system, VirtualBox is restricted to running 32-bit guest operating systems. Ensure any ISOs you download for lab work (like Alpine or Debian) are explicitly the 32-bit (x86/i386) versions.

Preload is consuming too much RAM

If you find that your active memory is constantly maxed out, preload might be caching too aggressively. You can tweak its behavior by editing /etc/preload.conf, or simply disable it entirely with sudo systemctl disable --now preload.

VirtualBox VMs run extremely slowly

Ensure that Hardware Virtualization (VT-x/AMD-V) is enabled in your computer's BIOS/UEFI. Even on older 32-bit processors, this feature is often present but disabled by default by the manufacturer.

Alpine Linux (Hardened)

Transform underpowered 64-bit hardware into an ultra-secure, Internet-facing web server using Alpine's hardened kernel and modern, memory-efficient services.

Why This Stack

  • Hardened Kernel: Alpine offers a linux-hardened kernel profile that adds unofficial grsecurity/PaX patches, significantly reducing the attack surface for an always-on home server.
  • Caddy Web Server: A modern, lightweight web server written in Go that automatically provisions TLS certificates via Let's Encrypt—eliminating the need to run separate, memory-heavy certbot instances.
  • tmpfs on /tmp: By mapping temporary files to RAM rather than disk, we prevent unnecessary write cycles, extending the life of aging mechanical hard drives or early-generation SSDs.
  • cronie: A reliable cron daemon for scheduled tasks that operates completely independently of systemd, preserving Alpine's low overhead.
  • musl libc Core: Ensures every base utility and installed service inherently consumes a fraction of the memory required by traditional glibc-based systems like Debian or Ubuntu.

Prerequisites

  • A 64-bit (x86_64) legacy PC with 512MB of RAM.
  • A blank USB Flash Drive (1GB or larger).
  • A secondary computer for downloading and flashing the ISO.
  • A wired Ethernet connection to your router for stable server connectivity.

Step 1: Download & Flash

Download the Alpine Linux Standard (x86_64) ISO. This image includes the necessary network drivers and the `setup-alpine` script for a fast bare-metal installation.

Download Alpine Standard (x86_64) ISO →

Flashing via Linux Terminal

bash
sudo dd if=alpine-standard-3.x.x-x86_64.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot the target machine from the USB drive.
  2. At the login prompt, log in as root.
  3. Run the system installer by typing: setup-alpine.
  4. Configure your keyboard layout, server hostname, and network interface (DHCP or static IP).
  5. Set a strong administrator (root) password.
  6. Select a timezone and allow the system to configure the NTP client.
  7. When prompted for a mirror, type 1 or f to find the fastest repository.
  8. Select your hard drive (e.g., sda) and type sys to execute a permanent installation.
  9. Type reboot and remove the installation media.

Step 3: Post-Install Optimization

Log in to your fresh installation to apply security and storage optimizations.

1. Install the Hardened Kernel

Swap out the default long-term support (LTS) kernel for the hardened variant to protect against memory corruption exploits.

bash
apk update apk add linux-hardened reboot

(Alpine will automatically update the extlinux bootloader to prioritize the newly installed hardened kernel).

2. Configure tmpfs for /tmp

Edit the filesystem table to route temporary files into RAM.

bash
echo "tmpfs /tmp tmpfs defaults,noatime,mode=1777 0 0" >> /etc/fstab mount -a

Step 4: Install the App Stack

Deploy the web server, SSH access, and scheduling daemon.

1. Install Caddy, Dropbear, and Cronie

bash
apk add caddy dropbear cronie rc-update add caddy default rc-update add dropbear default rc-update add cronie default

2. Basic Caddyfile Setup

Configure Caddy to serve a simple static site. Caddy automatically provisions HTTPS if you provide a valid domain name pointing to this server.

bash
mkdir -p /var/www/html echo "<h1>Secure Alpine Server</h1>" > /var/www/html/index.html nano /etc/caddy/Caddyfile

Modify the Caddyfile to look like this:

Caddyfile
:80 { root * /var/www/html file_server }

(To enable auto-HTTPS, simply change `:80` to your domain, e.g., `example.com`). Start the service: rc-service caddy start

3. Secure SSH (Dropbear)

It is highly recommended to disable password logins and use SSH keys for Dropbear.

bash
mkdir -p ~/.ssh nano ~/.ssh/authorized_keys

Paste your public key. Then, edit the Dropbear config to disable root password logins:

bash
sed -i 's/DROPBEAR_OPTS=""/DROPBEAR_OPTS="-s -g"/' /etc/conf.d/dropbear rc-service dropbear restart

Step 5: Verification

Confirm that your core services are running optimally.

Verify Caddy Version

bash
caddy version

Confirms the Caddy binary is correctly installed.

Check Active Listening Ports

bash
netstat -tuln

You should see Dropbear listening on port 22, and Caddy listening on port 80 (and 443 if you configured a domain).

Check System Memory Overhead

bash
free -h

Even with the hardened kernel and web server running, your RAM usage should be exceptionally low.

Troubleshooting

Caddy fails to start or acquire TLS

If you changed your Caddyfile to use a real domain for HTTPS, Caddy must be able to reach Let's Encrypt over ports 80 and 443. Ensure your home router is port-forwarding 80/443 directly to this server's internal IP address, and that your DNS A-record is correct.

Cannot SSH into the server

If you applied the Dropbear security options (-s -g), password logins are strictly forbidden. Ensure your private SSH key is correctly configured on your client machine. If you are locked out, you will need physical access to the server to revert /etc/conf.d/dropbear.

cron jobs are not executing

Verify that the cronie service is actually running using rc-service cronie status. Unlike systemd timers, traditional cron requires you to edit the crontab using crontab -e. Ensure your scripts have executable permissions (chmod +x).

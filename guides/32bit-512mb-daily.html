Tiny Core Linux

Revive ultra-low spec hardware with an OS that loads entirely into RAM, delivering absolute minimal overhead for web browsing and basic tasks.

Why This Stack

  • ZRAM Compression: Creates a compressed block device in RAM, essentially multiplying your usable memory space (crucial for 512MB systems).
  • ext2 Filesystem: Disables journaling to dramatically reduce read/write cycles, preserving the lifespan of aging IDE/SATA mechanical drives.
  • noop I/O Scheduler: The simplest I/O scheduler. On ancient hardware where the CPU is the bottleneck, stripping out complex scheduling logic yields faster disk access.
  • JWM (Joe's Window Manager): An extremely lightweight window manager that uses single-digit megabytes of RAM while providing a functional desktop experience.

Prerequisites

  • A 32-bit (x86) legacy PC with at least 512MB RAM.
  • A blank USB Flash Drive (1GB or larger).
  • A secondary computer to flash the ISO.
  • Wired Ethernet connection (recommended for initial setup).

Step 1: Download & Flash

Download the CorePlus ISO from the official Tiny Core Linux website. CorePlus includes the installation tools and wireless support needed for setup.

Download CorePlus ISO →

Flashing via Linux Terminal

bash
sudo dd if=CorePlus-current.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier, e.g., /dev/sdb. Do not append partition numbers like sdb1).

Step 2: Installation Walkthrough

  1. Insert the USB into the legacy PC and boot from it (you may need to press F2, F12, or DEL to change BIOS boot order).
  2. At the boot menu, select Boot Core Plus with Installation Extension.
  3. Once the desktop loads, click the tc-install icon on the bottom dock.
  4. Select your target hard drive (e.g., sda).
  5. Choose ext2 as the formatting option to save I/O cycles.
  6. In the boot options line, add: waitusb=5.
  7. Select "Core and X/GUI Desktop" as the installation type.
  8. Proceed and confirm the installation. Once finished, remove the USB and reboot.

Step 3: Post-Install Optimization

Boot into your fresh hard drive installation. Open the terminal to apply the optimization layer.

1. Configure the noop scheduler

Edit the bootloader configuration to force the noop scheduler.

bash
sudo vi /mnt/sda1/boot/extlinux/extlinux.conf

Append elevator=noop to the end of the APPEND line. Save and exit (:wq).

2. Install and Setup ZRAM

bash
tce-load -wi zram sudo modprobe zram num_devices=1 echo 268435456 | sudo tee /sys/block/zram0/disksize sudo mkswap /dev/zram0 sudo swapon /dev/zram0 -p 10

(This allocates 256MB for compressed ZRAM swap).

To make ZRAM persist across reboots, add the modprobe and swap commands to /opt/bootlocal.sh.

Step 4: Install the App Stack

Tiny Core uses tce-load to fetch and install applications. Run the following command to deploy your daily driver stack:

bash
tce-load -wi jwm sylpheed dillo
  • JWM: Ultra-lightweight window manager (replaces the default FLWM if desired).
  • Dillo: A fast, minimal graphical web browser. Cannot render modern JS heavy sites, but perfect for text/reading.
  • Sylpheed: A highly efficient, low-resource email client.

Step 5: Verification

Run these commands to ensure your optimizations are active.

Check I/O Scheduler

bash
cat /sys/block/sda/queue/scheduler

Output should show [noop] deadline cfq (noop bracketed means it is active).

Check ZRAM Allocation

bash
swapon -s

Output should list /dev/zram0 with the highest priority.

Troubleshooting

Installed apps disappear after reboot

Tiny Core loads entirely into RAM. To persist downloaded apps, ensure your /opt/tce/onboot.lst file contains the names of the packages. Use the GUI "Apps" tool -> Maintenance -> OnBoot Maintenance to manage this.

Custom scripts/settings are lost after reboot

Any changes made to files outside of /mnt/sda1 must be backed up before shutting down. Run filetool.sh -b in the terminal to trigger a backup of your personal settings to the persistent drive.

No Wi-Fi available

If you used the standard TinyCore ISO instead of CorePlus, you lack wireless drivers. Use an ethernet cable to run tce-load -wi wifi wireless_tools wpa_supplicant, then run sudo wifi.sh.

Fedora Workstation

Deliver a top-tier academic and development environment. With 4GB+ of RAM, Fedora's reference GNOME desktop, transparent compression, and telemetry-free development tools provide a professional-grade student workstation.

Why This Stack

  • Reference GNOME Experience: Fedora Workstation provides the purest, most stable implementation of the GNOME desktop environment, widely used in enterprise and academic settings.
  • btrfs + zstd Default: Fedora automatically formats drives with btrfs and zstd compression. This transparently shrinks large files (like lab VMs), extending the life of older SSDs and expanding usable capacity.
  • VSCodium: A community-driven, telemetry-free build of Microsoft VS Code. It provides all the necessary extensions and development features for Computer Science students without background tracking overhead.
  • VirtualBox: With 4GB of RAM, students can comfortably run 1GB-2GB lightweight virtual machines (like Alpine or minimal Debian) for networking, security, and OS coursework.
  • Anki & LibreOffice: The absolute best tools for spaced-repetition memorization and offline document creation, covering all standard academic requirements.

Prerequisites

  • A 64-bit (x86_64) PC with 4GB (or more) of RAM.
  • A blank USB Flash Drive (8GB or larger recommended).
  • Hardware Virtualization (VT-x/AMD-V) enabled in the computer's BIOS/UEFI.
  • A reliable internet connection for downloading the development stack.

Step 1: Download & Flash

Download the official Fedora Workstation Live ISO. This includes the full GNOME desktop environment and the graphical Anaconda installer.

Download Fedora Workstation ISO →

Flashing via Linux Terminal

bash
sudo dd if=Fedora-Workstation-Live-x86_64.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot from the USB drive and select Start Fedora Workstation.
  2. Once the GNOME desktop loads, click the Install to Hard Drive icon.
  3. Select your language and keyboard layout.
  4. Installation Destination: Select your target drive and leave the configuration on Automatic. Fedora will intelligently configure your btrfs subvolumes and ZRAM swap.
  5. Click Begin Installation.
  6. When the process finishes, reboot the system, remove the USB drive, and follow the on-screen prompts to create your student user account.

Step 3: Post-Install Optimization

Because Fedora natively configures ZRAM, btrfs, and systemd-oomd, our post-install focuses on adding the necessary third-party repositories for our development tools.

Add the VSCodium Repository

Import the GPG key and add the VSCodium repository to Fedora's package manager.

bash
sudo rpmkeys --import [https://gitlab.com/paulcarroty/vscodium-deb-rpm-repo/-/raw/master/pub.gpg](https://gitlab.com/paulcarroty/vscodium-deb-rpm-repo/-/raw/master/pub.gpg) printf "[gitlab.com_paulcarroty_vscodium_repo]\nname=download.vscodium.com\nbaseurl=[https://download.vscodium.com/rpms/](https://download.vscodium.com/rpms/)\nenabled=1\ngpgcheck=1\nrepo_gpgcheck=1\ngpgkey=[https://gitlab.com/paulcarroty/vscodium-deb-rpm-repo/-/raw/master/pub.gpg](https://gitlab.com/paulcarroty/vscodium-deb-rpm-repo/-/raw/master/pub.gpg)\nmetadata_expire=1h" | sudo tee -a /etc/yum.repos.d/vscodium.repo

Install CompSize (btrfs utility)

Install the compsize utility so we can actively measure how much space the btrfs filesystem is saving us.

bash
sudo dnf install compsize

Step 4: Install the App Stack

Install the educational suite, including LibreOffice, VirtualBox, VSCodium, and Anki.

bash
sudo dnf upgrade --refresh sudo dnf install @gnome libreoffice virtualbox vscodium anki

(Note: @gnome guarantees any missing core desktop packages are pulled in, though Workstation has most by default).

Configure VirtualBox Users

To allow the student user to manage VMs, map USB devices, and use bridged networking, add them to the vboxusers group.

bash
sudo usermod -aG vboxusers $USER

(The user must log out and log back in for this group assignment to take effect).

Step 5: Verification

Ensure that storage compression, memory limits, and critical development applications are functioning.

Verify btrfs Compression Savings

bash
sudo compsize /home

This will calculate and display the exact percentage of disk space saved by the native zstd compression.

Verify ZRAM Swap Configuration

bash
cat /proc/swaps

You should see a /dev/zram0 partition listed with the highest priority.

Verify Application Installations

bash
codium --version vboxmanage --version anki --version

Troubleshooting

VirtualBox Kernel Driver Not Loaded (vboxdrv)

If VirtualBox refuses to start VMs and complains about vboxdrv, Secure Boot is likely blocking the unsigned third-party kernel module. Either disable Secure Boot in your UEFI/BIOS, or install akmod-VirtualBox and sign the module manually using Fedora's MokManager.

Wayland Screen Sharing Issues

Fedora Workstation uses the Wayland display server by default. If a student is trying to screen share in Zoom or Discord and gets a black screen, log out, click the gear icon in the bottom right corner of the password screen, and select "GNOME on Xorg".

VSCodium Extensions Marketplace Missing Items

By default, VSCodium connects to the open-source Open VSX Registry, which may lack some proprietary Microsoft extensions (like Live Share or specific C# debuggers). Students can manually download .vsix files and install them directly within the editor if needed.

EndeavourOS + i3wm

Harness the bleeding-edge performance of Arch Linux via a friendly installer, combined with the ultra-efficient i3 tiling window manager to maximize screen space and minimize RAM overhead.

Why This Stack

  • EndeavourOS: Delivers the speed and rolling-release nature of Arch Linux but abstracts away the complex command-line installation, giving you the best of both worlds.
  • i3wm Tiling Layout: By dynamically arranging windows as tiles rather than overlapping them, i3 maximizes usable screen real estate (crucial for older, smaller laptops) and strips away the RAM consumed by heavy desktop environments.
  • systemd-oomd: A highly responsive Out-Of-Memory daemon that gracefully kills memory-hogging processes before they can cause a hard system freeze on 2GB machines.
  • BFQ Scheduler: The Budget Fair Queuing algorithm keeps the desktop highly responsive when dealing with mechanical hard drives under load.
  • Brave Browser: Features a powerful built-in ad and tracker blocker. By preventing heavy scripts from loading in the first place, it saves critical megabytes of RAM compared to standard browsers.

Prerequisites

  • A 64-bit (x86_64) PC with 2GB of RAM.
  • A blank USB Flash Drive (8GB or larger recommended).
  • A secondary computer for flashing the ISO image.
  • A stable internet connection (required for the EndeavourOS online installer).

Step 1: Download & Flash

Download the latest EndeavourOS ISO from their official website.

Download EndeavourOS ISO →

Flashing via Linux Terminal

bash
sudo dd if=EndeavourOS_latest.iso of=/dev/sdX bs=4M status=progress

(Replace /dev/sdX with your actual USB drive identifier. Do not append partition numbers).

Step 2: Installation Walkthrough

  1. Boot from the USB drive and select the default EndeavourOS boot option.
  2. Once the live environment loads, click Start the Installer (Online method recommended).
  3. Proceed through the Calamares installer steps (Location, Keyboard).
  4. Desktop Selection: When prompted to choose your desktop environment, select i3-wm. Uncheck any bloated software packages you don't need.
  5. Partitioning: Choose "Erase disk" (using ext4 is perfectly fine, or btrfs if you prefer snapshots). Ensure a Swap partition (with hibernation) or Swapfile is created.
  6. Create your user account, finish the installation, and reboot.

Step 3: Post-Install Optimization

Log in to your new i3 session. Open the terminal (usually Super + Enter) to apply Arch-specific optimizations.

1. Configure 50% ZRAM

Use Arch's zram-generator to compress 1GB of your physical memory.

bash
sudo pacman -S zram-generator sudo nano /etc/systemd/zram-generator.conf

Add the following configuration to the file:

ini
[zram0] zram-size = ram / 2 compression-algorithm = zstd

Reload the systemd daemon and start the ZRAM device:

bash
sudo systemctl daemon-reload sudo systemctl start systemd-zram-setup@zram0.service

2. Apply the BFQ I/O Scheduler

Force the disk queue to prioritize desktop responsiveness.

bash
echo bfq | sudo tee /sys/block/sda/queue/scheduler

(Optional: To make this persistent, create a udev rule or append `elevator=bfq` to your GRUB parameters).

Step 4: Install the App Stack

If you didn't install the core apps during the EndeavourOS setup, use pacman to fetch them now. Note: `brave-bin` is in the Arch User Repository (AUR), so you may need to use `yay` if `pacman` cannot find it.

bash
sudo pacman -Syu sudo pacman -S i3-wm alacritty thunar brave-bin

i3 Configuration Basics

Your i3 workflow is entirely defined by its configuration file. Open it to define your startup apps and keybindings.

bash
nano ~/.config/i3/config

Add your newly installed apps to autostart or bind them to keys. For example:

text
bindsym $mod+Return exec alacritty bindsym $mod+b exec brave bindsym $mod+f exec thunar

Reload i3 in place by pressing Super + Shift + C.

Step 5: Verification

Verify that your tiling manager and optimizations are active.

Check i3 Version

bash
i3 --version

Verify ZRAM

bash
zramctl

You should see /dev/zram0 listed with a size of roughly 1G and using the zstd algorithm.

Check Memory Constraints

bash
free -h

Review your idle RAM. With i3, it should sit comfortably under 400MB, leaving over 1.5GB for Brave and other tasks.

Troubleshooting

systemd-oomd is killing applications too aggressively

If your browser keeps closing unexpectedly, systemd-oomd might be acting too soon. You can adjust the pressure limits in /etc/systemd/oomd.conf, or rely primarily on your ZRAM and disable the service: sudo systemctl disable --now systemd-oomd.

pacman cannot find brave-bin

Because Brave is proprietary, it is hosted on the AUR rather than the official Arch repos. EndeavourOS ships with yay pre-installed. Run yay -S brave-bin as a standard user (do not use sudo with yay) to build and install the browser.

Screen tearing when scrolling or watching video

i3wm does not composite windows by default. Install the picom compositor (sudo pacman -S picom) and add exec --no-startup-id picom -b to your ~/.config/i3/config file, then reload i3.
